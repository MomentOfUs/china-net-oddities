# 🦞 中国互联网奇观档案 · 龙虾数据录入指南

各位“麻辣小龙虾”编辑在为档案馆添加或修改人物卷宗时，请务必严格遵守本指南。我们的目标是**把粗俗的网络小丑文化物化为“一级历史文物”进行学术解剖**，用冷冰冰的客观人类学笔调来衬托出互联网奇观最极致的荒诞与黑色幽默。

---

## 一、三大核心拟物玩法字段规范

### 1. 🗂️ 档案解密等级 (`clearance`)
每个奇观人物都需要判定并赋予一个**解密权限等级**（在 YAML 中配置为 `clearance: A/B/C/D`），以反映其在互联网社会重力场下的沉浮与宿命级别：

| 等级 | 档案馆分级名称 | 适用标本特征与判定标准 | 示例人物 |
| :--- | :--- | :--- | :--- |
| **D** | **D级解密 (日常娱乐)** | 纯粹由于算法推波助澜、作为大众日常消费快餐的娱乐型网红。 | 药水哥、阿Giao |
| **C** | **C级限阅 (底层挣扎)** | 涉及底层残酷生存危机、边缘地带挣扎，甚至有违法犯罪/法制纠纷的命运切片。 | 大力哥、窃·格瓦拉 |
| **B** | **B级绝密 (网络风暴眼)** | 已经脱离了个人意志，狂飙成为庞大的互联网抽象文化符号，遭受全网解构与审丑狂欢的人物。 | 孙笑川、马保国 |
| **A** | **A级封存 (封锁遗迹)** | 因政策、严重风波被全网物理封杀禁言，消逝在数字荒漠中，成为不可名状的“数字幽灵”。 | 铁山靠、秀才、郭老师 |

---

### 2. 🏷️ 时代“物证”陈列柜 (`relics`)
每一位人物需配置 **1-3 个“随案物证”**。在页面中，它们会被陈列在红木 specimen 标本柜中，鼠标移过会展示冷酷学术的解说词。

- **核心配方**：`name` (物证名称) + `desc` (博物馆解说词)
- **写作标准**：**“雅俗错位”**。用最冷冰冰、最高雅、最人类学/社会学的书面学术词汇，去解剖最粗鄙、最民间的网络名梗物。
- **写作公式**：`[物证No. XX · 拟物名称]：[发现/失窃/遗落于某地]，印证了[某种宏大或荒谬的时代浪潮]下的[微观个体宿命解构]。`
- **🚫 错误示范**：
  - *“剥好皮的玉米：阿giao拍视频时喜欢啃的玉米。”* —— 太过直白，毫无反差张力，缺乏阅读乐趣。
- **🟢 优秀示范**：
  - *“剥好皮的玉米：2017年采摘于河南许昌，该物证象征着在智能手机渗透率下沉前夕，华北平原农耕文明图腾与短视频审丑消费浪潮发生粗粝碰撞的物理媒介。”* —— 高雅的学术腔与土味剥玉米形成巨大张力，黑色幽默拉满。

---

### 3. 🔍 时代黑话边注 (`glossary`)
在编辑正文（`essay` 编者按 或 `comment` 档案注脚）时，凡是包含 glossary 列表中 `key` 的词语，前台页面中该词会自动带上虚线下划线。用户鼠标悬停时会弹出黄色便利贴进行科普。

- **核心配方**：`key` (正文需匹配的黑话词汇，须保持字形一致) + `val` (便利贴科普词条)
- **写作标准**：简短（1-2句），既能充当不懂网络史之人的科普，又需要对该词的起源和荒诞象征进行点睛式社会学拆解。
- **配置示例**：
  ```yaml
  glossary:
    - key: 打工是不可能打工的
      val: 周立齐被捕时的名言。后来演变为千万当代青年在面对职场重压时，通过躺平与消极抵抗来消解资本规训的标志性亚文化符号。
    - key: 一给我里giao giao
      val: 阿Giao标志性的拟音口号。它以反智、粗粝且极具节奏感的土味特质，成为2010年代末快手下沉审丑美学的集大成标签。
  ```

## 三、龙虾编辑发布工作流

1. **新建文件**：在本地 `src/content/phenomenon/` 目录下创建一个新的 `.yaml` 文件（例如 `mabaoguo.yaml`），按上方格式填写。
2. **放置图片**：
   - 将 360x360 比例正方形头像（建议进行复古做旧、偏灰偏暗过滤）保存到 `public/images/avatars/` 下，文件命名应与 yaml 文件名一致。
   - 大背景图放于 `public/images/heros/` 下。
3. **本地预览与测试**：
   - 打开终端，在项目根目录运行开发服务：
     ```bash
     pnpm run dev
     ```
   - 访问 `http://localhost:4321/`，检查首页的新卡片印章颜色是否正确、雷达图形状是否正常、以及点击进去后，正文黑话下划线浮现的便利贴是否有排版错乱。
4. **编译校验**：
   - 在提交发布前，必须进行打包构建测试，确保没有任何 schema 解析或格式错误：
     ```bash
     pnpm run build
     ```
   - 如果构建输出 `Complete!`，则证明数据录入无任何语法错误。
